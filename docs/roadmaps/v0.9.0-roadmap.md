# hf-fetch-model v0.9.0 Roadmap ‚Äî Safetensors Remote Inspection & Disk Usage

**Date:** March 23, 2026
**Status:** Proposed
**Supersedes:** v0.9.0-roadmap.md V1 (March 21, 2026) ‚Äî cache management scope moved to [v0.10.0](v0.10.0-roadmap.md)
**Context:** v0.8.x delivered cache-awareness (`list-files --show-cached`, `--dry-run` cache status, partial detection). Meanwhile, the **anamnesis** crate (tensor format transformation ‚Äî FP8, GPTQ, AWQ dequantization) now needs to read safetensors tensor metadata *before* downloading multi-GiB files. hf-fetch-model already has the HTTP Range infrastructure (chunked downloads) and the HF API integration ‚Äî it is the natural place for remote header fetching. v0.9.0 adds **safetensors remote header inspection** (the primary feature) and **`du`** (disk usage, nearly zero new code).

---

## Goals

1. **Safetensors header inspection** ‚Äî read tensor metadata (names, shapes, dtypes, offsets) from local cache or remote HF repos (via two HTTP Range requests), without downloading full files
2. **Disk usage (`du`)** ‚Äî measure what's taking space in the local HF cache

---

## Motivation: anamnesis Integration

anamnesis provides quantization scheme detection and dequantization. Its `parse_safetensors_header()` function (in `anamnesis::parse::safetensors`) classifies tensors by role (`Quantized`, `Scale`, `Passthrough`, `ZeroPoint`, `GroupIndex`) and detects the quantization scheme (`FineGrainedFp8`, `PerChannelFp8`, `PerTensorFp8`, `Gptq`, `Awq`, `Unquantized`).

Today, anamnesis requires the **entire file on disk** to parse a header. But the safetensors format stores the JSON header in the first bytes of the file ‚Äî two Range requests (8 bytes for the header length, then the header JSON) are enough to read it remotely. This means hf-fetch-model can fetch the raw header bytes from HuggingFace and anamnesis can interpret them, enabling workflows like:

- **Pre-download inspection:** "Is this model FP8 or GPTQ? What dtypes are the tensors?"
- **Selective download:** "Only download the model if it's a quantization scheme anamnesis supports"
- **Metadata cataloging:** "List all tensor names and shapes across a repo's safetensors files"

### Responsibility boundary

| Crate | Responsibility |
|-------|---------------|
| **hf-fetch-model** | Fetch raw header bytes (HTTP Range or local file), parse the safetensors JSON into tensor names/shapes/dtypes/offsets. No quantization-specific logic. |
| **anamnesis** | Interpret tensor metadata: classify roles, detect quantization scheme, find scale companions. Consumes hf-fetch-model's `TensorInfo` to build its own `TensorEntry` with `TensorRole`. |

hf-fetch-model does **not** depend on anamnesis. The dependency flows one way: anamnesis (or any consumer) calls hf-fetch-model's `inspect_safetensors()` and maps the result into its own types.

## Feature 1: Safetensors Remote Header Inspection

### Why

Safetensors files encode a JSON header in their first bytes: an 8-byte little-endian length prefix followed by the JSON blob. The header contains every tensor's name, dtype, shape, and byte offsets ‚Äî everything needed to understand a model's structure without downloading GiBs of weight data. Two HTTP Range requests suffice:

1. `Range: bytes=0-7` ‚Üí 8-byte `u64` header length (little-endian)
2. `Range: bytes=8-{8+length-1}` ‚Üí JSON header

### CLI

```
hf-fm inspect <REPO_ID> [FILENAME] [--revision <REV>] [--token <TOKEN>] [--cached] [--no-metadata] [--json]
```

| Flag | Effect |
|------|--------|
| `FILENAME` | Specific `.safetensors` file to inspect. If omitted, inspects all `*.safetensors` files in the repo. |
| `--revision` | Target branch, tag, or commit (default: `main`). |
| `--token` | HuggingFace API token (or `HF_TOKEN` env var). |
| `--cached` | Cache-only mode: read from local cache, fail if the file is not cached. Without this flag, inspect still **checks the cache first** and only makes HTTP Range requests if the file is not found locally. |
| `--no-metadata` | Suppress the `Metadata:` line in human-readable output. By default, `__metadata__` is printed when present (see below). Ignored with `--json`. |
| `--json` | Output the full `SafetensorsHeaderInfo` as JSON instead of the human-readable table. For programmatic consumption (e.g., anamnesis piping `hf-fm` output). |

**Default behavior (no `--cached`):** try local cache first, fall back to HTTP Range requests. This means `hf-fm inspect` is free and instant when the model is already downloaded ‚Äî no network round-trip at all. The `--cached` flag is for when you want to guarantee no network access (e.g., offline environments, CI).

### Implementation

#### Resolution order (cache-first)

```
1. Try local cache:
   a. hf_cache_dir() ‚Üí ~/.cache/huggingface/hub/
   b. models--{org}--{name}/refs/{revision} ‚Üí commit hash (via read_ref())
   c. models--{org}--{name}/snapshots/{commit}/{filename} ‚Üí local PathBuf
   d. If file exists: read first 8 bytes + header JSON from disk ‚Üí done (no network)
2. If not cached and --cached is not set: fall back to HTTP Range requests
   a. Build request URL: https://huggingface.co/{repo_id}/resolve/{revision}/{filename}
   b. GET with header `Range: bytes=0-7` ‚Üí 8-byte response ‚Üí parse as u64 (header_size)
   c. GET with header `Range: bytes=8-{8+header_size-1}` ‚Üí JSON header bytes
3. Deserialize JSON into tensor metadata
```

With `--cached`: step 1 only ‚Äî fail with an error if the file is not in the cache.

The HuggingFace CDN supports Range requests for all files (LFS and git-stored). For gated repos, the `Authorization: Bearer <token>` header is required on both requests.

#### JSON header format

The safetensors JSON header is an object where each key is a tensor name and each value has `dtype`, `shape`, and `data_offsets`:

```json
{
  "__metadata__": { "format": "pt" },
  "model.layers.0.self_attn.q_proj.weight": {
    "dtype": "F8_E4M3",
    "shape": [7168, 7168],
    "data_offsets": [0, 51380224]
  }
}
```

The `__metadata__` key (if present) is extracted separately and stored in `SafetensorsHeaderInfo::metadata`.

#### Cache resolution

The cache-first resolution logic lives in `inspect.rs` (not in `bin/main.rs`), since the library API `inspect_safetensors()` itself benefits from cache-first behavior ‚Äî any library consumer gets it for free. The separate `inspect_safetensors_local()` function takes a plain `&Path` for callers who already have a file path and don't need cache resolution.

### Library API

**Naming note:** hf-fetch-model uses `SafetensorsHeaderInfo` and `TensorInfo` (not `SafetensorsHeader` / `TensorEntry`) to avoid collision with anamnesis's own types of the same names. anamnesis's `SafetensorsHeader` includes quantization-specific fields (`scheme`, `gptq_config`, `awq_config`) that belong in anamnesis, not here.

**Serialization:** `SafetensorsHeaderInfo`, `TensorInfo`, and `ShardedIndex` derive `serde::Serialize` for `--json` output. `serde` is already a dependency (used by `repo.rs` for API response deserialization).

```rust
/// Metadata for a single tensor from a safetensors header.
///
/// This is hf-fetch-model's own type ‚Äî lightweight, no quantization logic.
/// Consumers (e.g., anamnesis) map this into their own richer types.
pub struct TensorInfo {
    /// Tensor name (e.g., `"model.layers.0.self_attn.q_proj.weight"`).
    pub name: String,
    /// Element dtype string as it appears in the header (e.g., `"F8_E4M3"`, `"BF16"`).
    pub dtype: String,
    /// Tensor shape (e.g., `[7168, 7168]`).
    pub shape: Vec<usize>,
    /// Byte offset range `[start, end)` within the data section of the file.
    pub data_offsets: (u64, u64),
}

impl TensorInfo {
    /// Total number of elements (product of shape dimensions).
    pub fn num_elements(&self) -> u64;

    /// Byte length of tensor data (`end - start`).
    pub fn byte_len(&self) -> u64;

    /// Bytes per element for the tensor's dtype, if recognized.
    ///
    /// Returns `None` for unknown dtype strings. Recognized dtypes:
    ///
    /// | Dtype string | Bytes | Notes |
    /// |-------------|-------|-------|
    /// | `"BOOL"` | 1 | |
    /// | `"U8"`, `"I8"` | 1 | |
    /// | `"F8_E4M3"`, `"F8_E5M2"` | 1 | FP8 variants |
    /// | `"U16"`, `"I16"`, `"F16"`, `"BF16"` | 2 | |
    /// | `"U32"`, `"I32"`, `"F32"` | 4 | |
    /// | `"U64"`, `"I64"`, `"F64"` | 8 | |
    ///
    /// This saves consumers from reimplementing the same lookup table.
    pub fn dtype_bytes(&self) -> Option<usize>;
}

/// Parsed safetensors header metadata.
pub struct SafetensorsHeaderInfo {
    /// All tensors in the header, in the order they appear in the JSON.
    pub tensors: Vec<TensorInfo>,
    /// Raw `__metadata__` entries, if present.
    ///
    /// For quantized models, this typically contains `quantize_config` entries
    /// (e.g., `bits`, `group_size`, `quant_method`) that consumers like anamnesis
    /// use to distinguish GPTQ from AWQ without downloading weights.
    pub metadata: Option<HashMap<String, String>>,
    /// Size of the JSON header in bytes.
    pub header_size: u64,
    /// Total file size in bytes (header + data), if known.
    ///
    /// **Source:** for local files, from `std::fs::metadata().len()`. For HTTP
    /// Range requests, extracted from the `Content-Range` response header of the
    /// first request (`bytes 0-7/TOTAL` ‚Üí `TOTAL`). This is free ‚Äî no extra
    /// request needed.
    pub file_size: Option<u64>,
}

impl SafetensorsHeaderInfo {
    /// Total parameter count across all tensors.
    pub fn total_params(&self) -> u64;

    /// Returns tensors matching a dtype string (e.g., `"F8_E4M3"`).
    pub fn tensors_with_dtype(&self, dtype: &str) -> Vec<&TensorInfo>;
}

/// Inspects a single safetensors file's header (cache-first).
///
/// Checks the local HF cache first. If the file is cached, reads the header
/// from disk with zero network requests. Otherwise, falls back to two HTTP
/// Range requests (8-byte length prefix + JSON header). Does not download
/// tensor data in either case.
///
/// # Errors
///
/// Returns [`FetchError::Http`] if the Range requests fail.
/// Returns [`FetchError::SafetensorsHeader`] if the header is malformed.
pub async fn inspect_safetensors(
    repo_id: &str,
    filename: &str,
    token: Option<&str>,
    revision: Option<&str>,
) -> Result<SafetensorsHeaderInfo, FetchError>;

/// Inspects a single safetensors file's header from a local file path.
///
/// Reads the first `8 + header_size` bytes from disk. Does not read tensor data.
///
/// # Errors
///
/// Returns [`FetchError::Io`] if the file cannot be read.
/// Returns [`FetchError::SafetensorsHeader`] if the header is malformed.
pub fn inspect_safetensors_local(
    path: &Path,
) -> Result<SafetensorsHeaderInfo, FetchError>;

/// Inspects all `*.safetensors` files in a repository (cache-first per file).
///
/// Fetches the file listing via `list_repo_files_with_metadata()`, then
/// inspects each `.safetensors` file's header via [`inspect_safetensors()`].
/// For each file, checks the local cache first and only makes HTTP Range
/// requests on cache miss. Returns full per-shard headers in filename order.
///
/// For a lightweight summary of sharded models (tensor counts per shard
/// without fetching individual headers), use [`fetch_shard_index()`] instead.
/// The CLI uses `fetch_shard_index()` as a fast path when no filename is
/// given and an index file exists.
///
/// # Errors
///
/// Returns [`FetchError::Http`] if the metadata or Range requests fail.
pub async fn inspect_repo_safetensors(
    repo_id: &str,
    token: Option<&str>,
    revision: Option<&str>,
) -> Result<Vec<(String, SafetensorsHeaderInfo)>, FetchError>;
```

### New error variant

```rust

SafetensorsHeader { filename: String, reason: String },
```

### New module: `inspect.rs`

All safetensors inspection logic lives in a new `src/inspect.rs` module:
- `resolve_cached_path()` ‚Äî cache-first resolution: `hf_cache_dir()` + `read_ref()` + snapshot directory
- `fetch_header_bytes()` ‚Äî HTTP Range request helper (two requests); extracts `file_size` from `Content-Range` header
- `parse_header_json()` ‚Äî JSON deserialization into `Vec<TensorInfo>` + metadata
- `fetch_shard_index()` ‚Äî fetch and parse `model.safetensors.index.json` (cache-first)
- `inspect_safetensors()`, `inspect_safetensors_local()`, `inspect_repo_safetensors()` ‚Äî public API

### Sharded model optimization

Many large models are sharded (e.g., DeepSeek-V3 has 163 `.safetensors` files). Naively inspecting every shard means 2√ó163 = 326 HTTP Range requests. But sharded models include a `model.safetensors.index.json` file that maps every tensor name to its shard filename ‚Äî all without touching the actual safetensors files. (The index does *not* contain shapes, dtypes, or byte offsets ‚Äî only the tensor-to-shard assignment.)

The **CLI** uses the shard index as a fast path:

1. No `FILENAME` given: CLI calls `fetch_shard_index()` first. If an index exists, display the summary table (tensor count per shard) from the index alone ‚Äî **1 request** instead of 326. No Range requests needed.
2. No `FILENAME` given, no index: CLI falls back to `inspect_repo_safetensors()`, which fetches full headers for each shard via concurrent Range requests.
3. `FILENAME` given: CLI calls `inspect_safetensors()` directly ‚Äî Range requests for the single shard's full header.

The **library** keeps the APIs separate: `fetch_shard_index()` returns a `ShardedIndex` (lightweight, index-only), while `inspect_repo_safetensors()` always returns full `SafetensorsHeaderInfo` per shard (expensive but complete). Callers like anamnesis choose which to call based on their needs.

```rust
/// Parsed `model.safetensors.index.json` for a sharded model.
pub struct ShardedIndex {
    /// Mapping from tensor name to shard filename.
    pub weight_map: HashMap<String, String>,
    /// Ordered list of shard filenames.
    pub shards: Vec<String>,
    /// Raw metadata from the index, if present.
    pub metadata: Option<HashMap<String, serde_json::Value>>,
}

/// Fetches and parses the shard index for a sharded safetensors model (cache-first).
///
/// Returns `Ok(None)` if the repo has no `model.safetensors.index.json` (i.e.,
/// the model is not sharded or uses a single safetensors file).
///
/// # Errors
///
/// Returns [`FetchError::Http`] if the index fetch fails.
/// Returns [`FetchError::SafetensorsHeader`] if the index JSON is malformed.
pub async fn fetch_shard_index(
    repo_id: &str,
    token: Option<&str>,
    revision: Option<&str>,
) -> Result<Option<ShardedIndex>, FetchError>;
```

### Concurrency for multi-file inspection

When the shard index shortcut doesn't apply (no index file, or single-file mode), headers are fetched concurrently with a semaphore (default: 4 concurrent requests). Each header is small (typically 10-100 KiB), so this completes in seconds even for repos with 100+ shards.

## Feature 2: `hf-fm du [REPO_ID]`

### Why

`~/.cache/huggingface/hub/` grows unbounded. Users need to know what's taking space before they can decide what to prune (in v0.10.0).

### Behavior

Without arguments: all cached repos, sorted by size descending. With a repo: per-file breakdown. Repos with incomplete downloads show a `PARTIAL` marker.

### Implementation

**Summary mode (no args):** Calls existing `cache::cache_summary()`, sorts by `total_size` descending, formats with `format_size()`. Almost zero new code.

**Per-repo mode:** Walks `<cache_dir>/models--<org>--<name>/snapshots/<commit>/` and lists each file with its size. Reuses `count_snapshot_files()` logic but returns per-file entries instead of totals.

## Non-goals

| Feature | Why not in hf-fetch-model | Where it belongs |
|---------|--------------------------|-----------------|
| Grouped tensor output (e.g., clustering `.qweight`/`.qzeros`/`.scales` triplets) | Requires quantization-specific knowledge (GPTQ vs AWQ grouping rules) | anamnesis CLI (`amn inspect`) ‚Äî consumes hf-fetch-model's `TensorInfo` and applies its own `TensorRole` classification |
| Quantization scheme detection from tensor patterns | Domain logic, not transport/parsing | anamnesis `parse_safetensors_header()` |
| Dtype enum in hf-fetch-model | Would couple hf-fetch-model to the safetensors spec's dtype list; consumers (anamnesis, candle) have their own enums | `dtype` stays as `String`; `dtype_bytes()` helper covers the common need |

## Implementation Order

| Order | Feature | Rationale |
|-------|---------|-----------|
| 1 | `du` | Simplest ‚Äî reuses `cache_summary()` almost entirely. Ships immediate value while `inspect` is developed. |
| 2 | `inspect` (single file, remote) | Core feature. Implement `fetch_header_bytes()` + `parse_header_json()` + `inspect_safetensors()`. Validates the Range request approach. |
| 3 | `inspect` (cache-first + `--cached`) | Add cache resolution to `inspect_safetensors()`, add `inspect_safetensors_local()`. Reuses the JSON parser from step 2. |
| 4 | `inspect` (all files in repo) | `inspect_repo_safetensors()` with concurrent fetching. Composes the single-file API from step 2 with `list_repo_files_with_metadata()`. |
| 5 | `inspect` (shard index fast path) | Add `fetch_shard_index()` library API + CLI fast path: when no filename is given and an index exists, display summary from index (1 request) instead of fetching all shard headers. |

## Appendix A: Dogfooding Feedback

The following enhancements were identified while using `inspect` during the **anamnesis** BnB (BitsAndBytes) quantization work (March 2026). Both are `inspect`-related and are candidates for inclusion in v0.9.x or v0.10.0.

### A.1 `inspect --filter <PATTERN>` ‚Äî tensor name filtering

**Use case:** "Show me just layer 0 to understand the model structure" ‚Äî currently requires piping through `grep`, which works but loses the header/summary lines and table formatting.

```
hf-fm inspect Qwen/Qwen3-4B --filter "layers.0"
```

**Implementation:** filter the `Vec<TensorInfo>` by glob or substring match before display. Library API could add an optional `filter` parameter or a post-parse helper. Low effort ‚Äî the data is already in memory.

### A.2 `inspect --diff <REPO1> <REPO2>` ‚Äî tensor layout comparison

**Use case:** "Is this NF4 model the same tensor structure as the FP8 variant?" ‚Äî currently requires inspecting both models separately and manually diffing the output.

```
hf-fm inspect --diff RedHatAI/Llama-3.2-1B-Instruct-FP8 casperhansen/llama-3.2-1b-instruct-awq
```

**Output would show:**
- Tensors only in model A (e.g., `weight_scale_inv` tensors present in FP8 but not AWQ)
- Tensors only in model B (e.g., `qweight`, `qzeros`, `scales` triplets in AWQ but not FP8)
- Tensors in both with different dtypes or shapes
- Tensors in both that match exactly

**Implementation:** inspect both repos, align tensors by name, classify into the four buckets above. Higher effort than `--filter` but unique ‚Äî no existing tool provides this. Particularly valuable for format research and anamnesis development.

### B.3 Python `hf` CLI quirks discovered during testing

| Quirk | Detail |
|-------|--------|
| `--include` silently ignored | `hf download <repo> --include "*.safetensors"` warns and ignores the flag on older versions (v1.3.5) ‚Äî downloads only non-LFS files |
| Symlink warnings on Windows | Every download emits a long warning about symlinks not being supported, suggesting Developer Mode activation |
| Version nag on every invocation | `hf` prints an upgrade notice on stdout, polluting piped output |

These quirks highlight areas where `hf-fm` already provides a better experience (correct `--filter`/`--preset`, no symlink dependency, clean output).
